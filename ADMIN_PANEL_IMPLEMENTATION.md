# Реализация админ панели для Telegram бота

## Обзор

Создана полнофункциональная админ панель для управления Telegram чатами через веб-интерфейс. Панель реализована как Telegram Mini App с правами доступа на основе базы данных.

## Структура файлов

### Основные файлы:
- `api.py` - Основной API сервер с эндпоинтами для админ панели
- `admin/client/index.html` - Главная страница со списком админских чатов
- `admin/client/chat_admin.html` - Админ панель конкретного чата
- `admin/server/api.py` - Отдельный сервер админ панели (опционально)
- `admin/server/run.py` - Скрипт запуска сервера админ панели
- `admin/README.md` - Подробная документация

### Вспомогательные файлы:
- `check_db.py` - Проверка структуры базы данных
- `create_test_data.py` - Создание тестовых данных
- `test_admin_db.py` - Тестирование базы данных

## Функциональность

### 1. Главная страница админ панели (`index.html`)
- Авторизация через Telegram Web App
- Отображение списка чатов, где у пользователя есть админские права
- Показ аватаров чатов через Telegram Bot API
- Количество доступных прав для каждого чата
- Переход на админ панель конкретного чата

### 2. Админ панель чата (`chat_admin.html`)
- Отображение информации о чате
- Список доступных функций в зависимости от прав
- Просмотр пользователей чата (если есть право `can_see_users`)
- Красивый интерфейс с анимациями и переходами

### 3. API эндпоинты

#### `/user-admin-chats/{user_id}`
- Возвращает список чатов, где пользователь имеет админские права
- Включает информацию о правах для каждого чата

#### `/chat-admin-panel/{chat_id}/{user_id}`
- Возвращает данные для админ панели конкретного чата
- Проверяет права пользователя
- Возвращает список доступных функций

#### `/chat-users/{chat_id}`
- Возвращает список пользователей чата
- Включает информацию о рангах и активности

#### `/chat-avatar/{chat_id}`
- Получает аватар чата через Telegram Bot API

## Права доступа

Система поддерживает 5 типов прав:

1. **can_see_users** - Просмотр пользователей чата
2. **can_do_admin** - Административные функции
3. **can_recom** - Управление рекомендациями
4. **can_links** - Управление ссылками
5. **can_dk** - Специальные DK функции

## База данных

### Таблица `admins` в `admin.db`:
```sql
CREATE TABLE admins (
    user_id INTEGER,
    chat_id INTEGER,
    chat_name TEXT,
    can_see_users INTEGER DEFAULT 0,
    can_do_admin INTEGER DEFAULT 0,
    can_recom INTEGER DEFAULT 0,
    can_links INTEGER DEFAULT 0,
    can_dk INTEGER DEFAULT 0,
    PRIMARY KEY (user_id, chat_id)
)
```

## Запуск
